# AGENTS.md

AI エージェントがこのリポジトリで作業する際の非自明なルール集。設定ファイルから読み取れる情報（インデント、フレームワーク等）は省略している。

## 必須: コード変更後

- **`make ci` を実行**し、エラーをすべて解消してからコミットすること
- `make ci` 後にファイル差分があると CI が落ちる（dirty check）。フォーマットは **事前に `pnpm run format`** で適用しておくこと

## フォーマッター

- `.astro` ファイル → **Prettier**
- それ以外 → **Biome**
- 混同すると CI が壊れる

## TypeScript

- **`@ts-ignore` 禁止** — カスタムスクリプトで検出される。代わりに `@ts-expect-error` を使うこと
- **`any` 禁止** — Biome の `noExplicitAny: error` で検出される

## 生成ファイル

- `src/data/author-status.json` は自動生成ファイル。**手動編集しない**

## 意図コメント（`[Intended]` / `[Policy]` / `[Workaround]`）

一見するとバグ・非効率・オーバーエンジニアリングに見えるが、意図的にそう実装しているコードには以下のタグコメントが付与されている。

- `// [Intended] 理由` — 仕様・設計上の意図がある。リファクタリングや最適化の対象にしないこと
- `// [Policy] 理由` — 運用ルール・組織方針に基づく制約。ルール自体が変更されない限り維持すること
- `// [Workaround] 理由` — 外部バグやライブラリ制約の回避策。対象の制約が解消されたら除去すること

### エージェントへの指示

1. これらのタグが付いたコードを変更・削除・リファクタリングしてはならない
2. 変更が必要な場合は、まずコメントに記載された理由を確認し、その理由がまだ有効かを判断すること
3. 新たにコードを書く際、上記に該当する実装を行った場合は適切なタグコメントを付与すること

## テキストの大文字・小文字表示

- **`text-transform: uppercase` を新たに追加しない** — テーマ全体でテキストを強制大文字にしない方針
- **例外**: `src/styles/portal.css` の `.floating-ui-pill .lang-btn` のみ `text-transform: uppercase` を維持する（言語切替ボタンの視認性のため）
- CSS 変数 `--category-text-transform` は `none` が正しい値。`uppercase` に変更しないこと

## コミット・コメント規約

- コメント、テスト記述、コミットメッセージは**日本語**
- **Conventional Commits** に従うこと（Release Please が使用している）
